NoSQL

Nerelacione
Manje striktna struktura i veze
Fleksibilnije strukture.

Ima vise tipova:

Document store: cuvanje u dokumentima tipa json (MongoDB), koristi se kod CMS-a
Key-value store: samoobjasnjivo (Redis), koristi se kod API-a
Column-Family stores: podaci se cuvaju u kolonama umesto u redovima (Cassandra), koristi se kod event trackinga i time series-a
Graph databases: graf struktura, veze jake, i veze i nodeovi imaju svojstva (Neo4j), koristi se za drustvene mreze

Jezici: MongoDB, Sypher, CQL...
Skalabilnost: vertikalna (dodavanje jos servera)

JOINOVANJE

SELECT brand, model, price, sold, sold_price 
FROM sold_cars SC
LEFT / RIGHT / FULL / INNER JOIN cars C ON SC.cars_id = C.id;

SELECT
	S.name,
	S.role,
	D.city
FROM sold_cars SC
	FULL JOIN staff S ON SC.seller = S.id
	LEFT JOIN dealerships D ON S.dealership_id = D.id
WHERE SC.id IS NULL
	AND S.role = 'Salesperson';
