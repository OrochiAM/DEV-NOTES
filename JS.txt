HTTP Request

Komponente:
1. Path(URL)
2. Method(GET, POST, PUT, DELETE, DRUGE(PATCH, OPTIONS))
3. Body(Opciono)
4. Headers(Metapodaci o requestu(npr tip browsera), ima ih mnogo)

https:// - scheme

Path
gde zivi resurs
BaseURL - deo URL-a koji se ne menja
Endpoint - deo koji se menja

Svaki klik na link je GET request

Methods
GET - get data
POST - send new data (ne vraca podatke)
PUT - modifikacija postojecih podataka (update)
DELETE - samoobjasnjivo

    })

REST 
Representational State Transfer

Design pattern za komunikaciju klijenta i servera.

Principi:
Separation of Client and Server, server vraca JSON podatke i nije ga briga kako ga koristi klijent. (server ne renderuje)

Statelessness, kada klijent napravi request, ne cuva se u memoriji (zaboravi ga), ako klijentu trebaju te informacije uvek treba sve slati.

Accessing Resources - koriscenje standardizovanih URL endpointa

/bikes endpoint imenica -> vraca se i postavlja se niz objekata tipa imenice (GET, POST)

/bikes/:id (/bikes/id) -> specifican objekat (GET, PUT, DELETE)
:id - URL parametar 

Nested
/bikes/:id/reviews -> vraca niz review-ova za bike id
/reviews - nested resource

function filterArray(array, callback) {
    const resultingArray = []
  
    for (let item of array) {
        const shouldBeIncluded = callback(item)
        if (shouldBeIncluded) {
            resultingArray.push(item)
        }
    }
    return resultingArray
}
